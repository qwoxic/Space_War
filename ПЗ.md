# Пояснительная записка  
## к проекту «Space War»

---

## 1. Введение

Проект **«Danger Monsters»** представляет собой 2D-игру в жанре аркадного экшена, разработанную на языке программирования Python с использованием графической библиотеки **Arcade**.

Цель проекта — создание простого, но функционального игрового приложения, демонстрирующего навыки:
- объектно-ориентированного программирования;
- работы с графическим интерфейсом;
- организации игрового цикла;
- хранения данных с использованием базы данных SQLite.

Проект выполнен в рамках учебной программы **Яндекс Лицея**.

---

## 2. Назначение проекта

Приложение предназначено для:
- демонстрации принципов разработки игр на Python;
- закрепления знаний по архитектуре программ;
- изучения работы с событиями клавиатуры и мыши;
- работы с базой данных для сохранения результатов.

Игра может использоваться как учебный пример или как простая аркадная игра для пользователей.

---

## 3. Используемые технологии

В проекте использованы следующие технологии и инструменты:

- **Python 3** — основной язык программирования.
- **Arcade** — библиотека для создания 2D-игр.
- **SQLite** — встроенная база данных для хранения результатов.
- **Object-Oriented Programming (OOP)** — организация логики игры.
- **Virtual Environment (venv)** — изоляция зависимостей проекта.

## 5. Описание файлов проекта

### 5.1 `main.py`

Файл запуска приложения.

Функции:
- создание игрового окна;
- установка размеров экрана;
- запуск главного цикла Arcade;
- отображение стартового меню.

---

### 5.2 `menu.py`

Отвечает за пользовательский интерфейс меню.

Реализует:
- стартовое меню игры;
- обработку нажатия клавиш и мыши;
- переход к игровому экрану;
- выход из приложения;
- отображение таблицы рекордов.

---

### 5.3 `game.py`

Основной файл игровой логики.

Содержит:
- игровой цикл (`on_update`, `on_draw`);
- управление персонажем;
- логику врагов и босса;
- обработку столкновений;
- подсчёт очков;
- завершение игры и переход на финальный экран.

---

### 5.4 `player.py`

Реализует игровой персонаж.

Функциональность:
- хранение позиции персонажа;
- управление движением;
- хранение и обработка здоровья;
- получение урона.

---

### 5.5 `enemy.py`

Содержит логику обычных врагов.

Реализует:
- движение врагов в сторону игрока;
- параметры здоровья и скорости;
- обработку уничтожения врагов.

---

### 5.6 `boss.py`

Файл логики финального босса.

Содержит:
- несколько фаз поведения;
- уникальные атаки;
- систему снарядов;
- изменение сложности в зависимости от здоровья.

---

### 5.7 `levels.py`

Отвечает за систему уровней.

Реализует:
- хранение конфигураций уровней;
- подсчёт уничтоженных врагов;
- переход между уровнями;
- определение финального уровня с боссом.

---

### 5.9 `database.py`

Реализует работу с базой данных SQLite.

Функциональность:
- создание базы данных при первом запуске;
- сохранение результатов игры;
- хранение имени игрока, очков и даты;
- получение таблицы рекордов.

---

## 6. Работа с базой данных

Для хранения результатов используется база данных **SQLite**.

### Таблица `scores` содержит поля:
- `id` — уникальный идентификатор записи;
- `player_name` — имя игрока;
- `score` — количество очков;
- `date` — дата прохождения.

База данных автоматически создаётся в папке `data` при первом запуске игры.

---

## 7. Управление в игре

- **W** — движение вверх  
- **A** — движение влево  
- **S** — движение вниз  
- **D** — движение вправо  
- **ЛКМ / Пробел** — атака  
- **ESC** — выход в меню  

---

## 8. Инструкция по запуску

1. Установить Python версии **3.10 и выше**.
2. Создать виртуальное окружение:
python -m venv .venv

3. Активировать виртуальное окружение:
- Windows:
  ```
  .venv\Scripts\activate
  ```
4. Установить зависимости:
pip install arcade

5. Запустить проект:
python main.py
