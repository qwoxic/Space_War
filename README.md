# ТЕХНИЧЕСКОЕ ЗАДАНИЕ
## Проект: Danger Monsters

## 1. Общая информация
- **Название:** Danger Monsters
- **Жанр:** Top-down шутер с элементами survival и головоломками
- **Платформа:** ПК (Windows, macOS, Linux)
- **Технологии:** Python 3.10+, библиотека Arcade 2.6+, SQLite для хранения данных

## 2. Основной геймплей
### 2.1. Управление
- **Движение:** клавиши WASD или стрелки
- **Стрельба:** левая кнопка мыши (прицеливание курсором)
- **Взаимодействие:** клавиша E (открытие дверей, сбор предметов)
- **Пауза:** клавиша ESC
- **Использование предметов:** цифры 1-4

### 2.2. Персонаж игрока
- Здоровье (100 HP), возможность восстановления через аптечки
- Основное и альтернативное оружие
- Инвентарь для временных улучшений

### 2.3. Система врагов
**Типы противников:**
- **Мясник** – ближний бой, быстрое перемещение
- **Стрелок** – атакует снарядами с расстояния
- **Танк** – высокий запас здоровья, медленное движение
- **Колдун** – финальный босс с уникальными способностями

**ИИ врагов:**
- Состояния «патруль», «преследование», «атака»
- Волновая система: увеличение количества и сложности врагов с каждой новой волной

### 2.4. Уровень и окружение
**Процедурная генерация:**
- 3 связанные комнаты с переходами

**Объекты на уровне:**
- Стены и препятствия
- Двери, открывающиеся после очистки волны
- Тайники и скрытые проходы

**Загадки:**
- Поиск ключа в комнате
- Активация плиток в определённой последовательности
- **Награда:** временные улучшения или доступ к скрытым зонам

## 3. Технические системы
### 3.1. Графика и отображение
- Разрешение: 1024×768 пикселей (базовое), поддержка других разрешений
- Камера, следующая за игроком
- Анимации: движение персонажа, атаки, смерть врагов, эффекты выстрелов
- Миникарта в углу экрана

### 3.2. Звук
- Фоновая музыка (разная для меню и уровней)
- Звуковые эффекты: выстрелы, попадания, шаги, активация предметов
- Регулировка громкости через меню настроек

### 3.3. Хранение данных
**База данных:** SQLite (файл `game.db`)

**Таблицы:**
- `highscores` (имя, очки, волна, время выживания, дата)
- `settings` (громкость, настройки управления, графика)
- `player_stats` (убийства, точность, решённые загадки)

**Автосохранение:** каждые 5 минут и при выходе

### 3.4. Интерфейс
**Главное меню:**
- Кнопки «Играть», «Рекорды», «Настройки», «Выход»

**Игровой HUD:**
- Шкала здоровья
- Счётчик текущей волны и очков
- Отображение активных улучшений
- Индикатор патронов/перезарядки

**Меню паузы:**
- «Продолжить», «Сохранить», «Настройки», «Выйти в меню»

**Экран Game Over:**
- Итоговая статистика
- Кнопка рестарта
- Переход в таблицу рекордов

## 4. Критерии завершённости
### 4.1. Обязательные компоненты
1. Рабочее управление персонажем и стрельба
2. 3 типа врагов с разным поведением (Мясник, Стрелок, Танк)
3. Волновая система (минимум 10 волн)
4. 3 комнаты с переходами и простыми загадками
5. Финальный босс «Колдун»
6. Система сохранения рекордов в SQLite
7. Полный цикл интерфейса (меню, HUD, пауза, Game Over)
8. Звуковое сопровождение (музыка + основные эффекты)

### 4.2. Дополнительные возможности (по желанию)
- Более сложные загадки и скрытые зоны
- Разные виды оружия и улучшений
- Система достижений
- Настройка управления
- Частицы и визуальные эффекты

## 5. Технические требования
**Минимальные:**
- Процессор: 1.5 GHz
- ОЗУ: 2 GB
- Видеокарта: с поддержкой OpenGL 3.3+
- 100 MB свободного места на диске

**Рекомендуемые:**
- Процессор: 2.0 GHz+
- ОЗУ: 4 GB
- Видеокарта: с поддержкой OpenGL 4.0+
